基于Hexo和ParticleX主题搭建个人博客

基于Hexo和ParticleX主题搭建个人博客

前言

本站基于Argvchs大佬的Hexo 博客搭建教程建成。
结合一下我自己建站的经历和建站时遇到的一些问题，作此教程，希望能够为大家带来帮助。
推荐工具:

  • GitHub有时登陆不上，Windows推荐工具FastGithub,在右方的release处下载（带exe可执行文件)
  • 在要访问GitHub的时候，打开工具，即可快速访问GitHub。

一.本地部署个人博客

1.搭建环境

软件要求

  • Git 推荐 阿里镜像(Git官方下载较慢)
  • Node.js Node.js中文网
  • 域名(可不选，机构审核可能需要几天时间，建议提前做好准备)

检验环境是否安装成功

win + R ，输入CMD打开命令行界面,分别输入下方命令检测是否安装成功(有版本号即安装成功)。

git --version
node -v

2.开启Corepack(可选,Node.js安装伴随npm)

启用Corepack后自动安装Yarn和PNPM(包管理器，便于下载资源)
启用命令
提示:前往目录C:\Windows\System32以管理员身份打开cmd.exe(命令行界面)

corepack enable

关于npm、pnpm、Yarn以及Yarn PnP的速度效率Argvchs大佬在Hexo 博客搭建Ⅰ进行了比较，感兴趣的朋友可前去自行观看。

3.安装Hexo

新建个人博客文件夹(博客相关文件存储于此),路径不要有中文。
进入该文件夹，鼠标右键空白处，选择 Git Bush Here
输入下列命令，安装并初始化Hexo 

#安装Hexo
pnpm add hexo-cli -g
#初始化Hexo
hexo init --no-install
pnpm i

4.查看效果

开启博客服务器

#生成博客相关页面的静态文件，g即generate(生成)。
hexo g
#本地运行博客，s即server(服务器)。
hexo s

开启后，浏览器输入网址 http://localhost:4000/ 进行查看
(默认开启端口为4000，若想更改,则在运行博客指令后加上 -p 端口号,例如,hexo s -p 8080,即在8080端口运行)

若成功显示Hexo界面，则取得阶段性胜利,加油加油再接再厉！

5.安装主题(让博客更好看的魔法)

5.1首先选择心仪的主题

点我传送 (Windows访问GitHub推荐工具FastGithub)

提示:

  • 点击图片查看主题样式
  • 点击下的主题名进入Github项目页面,点击绿色的Code按钮，选择Clone下的HTTPS,复制项目路径

5.2安装主题

找到博客的根目录(第3步创建的博客文件夹),进入themes文件夹
鼠标右键空白处，选择 Git Bush Here
输入下列命令

#将别人上传的主题文件下载到自己博客文件夹的themes目录中
git clone <复制的内容> <复制的主题的名字> --depth=1

5.3配置网页参数

打开根目录下的_config.yml文件

找到theme:
将后面内容改为自己主题的名字
例如:

theme: particlex

更多参数如下

title: # 标题
subtitle: # 副标题
description: # 描述
keywords: # 关键字
author: # 作者

根据自己的想法以及所选主题的要求修改参数即可

更多参数参考Hexo为文档

举个栗子(安装ParticleX主题)

我的主题为:ParticleX

在themes文件夹下输入命令

#将ParticleX主题下载到themes中
git clone https://github.com/argvchs/hexo-theme-particlex.git particlex --depth=1

安装完成后
打开博客根目录下的-config.yml文件,找到theme并将其参数设置为ParticleX

theme: particlex
创建ParticleX主题特殊页面(分类页面、标签页面、关于(about)页面)

创建分类页面

  • 创建页面
#创建页面命令:hexo new page 页面名称
hexo new page categories
  • 添加分类
    打开 source/categories/index.md
    在两条 --- 之间添加 type: categories
    注意:type后的:和categories之间有空格

创建标签页

  • 创建页面
#创建页面命令:hexo new page 页面名称
hexo new page tags
  • 添加分类
    打开 source/tags/index.md
    在两条 --- 之间添加 type: tags
    注意:type后的:和categories之间有空格

创建关于页

  • 创建页面
#创建页面命令:hexo new page 页面名称
hexo new page about
  • 添加内容
    打开 source/tags/index.md
    在两条 --- 之后添加自己想要添加的内容

二.将博客部署到Github Pages上

1.配置SSH密钥(实现本地代码与Github代码库同步)

前往GitHub的个人主页(若无则先创建账号)

点击New新建仓库,名称为<Github用户名>.github.io,然后点击Create repositpory完成创建

在个人博客根目录下，右键鼠标，选择Git Bash Here

#输入完后，按回车(可能4次就行)即可
ssh-keygen -t rsa -C <你的邮箱>

添加到SSH-Agent

eval "$(ssh-agent -s)"
ssh-add ~/.ssh/id_ed25519

拷贝公钥

#该公钥一般存在 C 盘 user 中，此命令失败的话可以直接找到该文件并且用记事本打开复制
clip < ~/.ssh/id_rsa.pub

在Github中添加你的公钥

  • 登录Github，进入settings
  • 点击SSH and GPG Keys
  • 点New SSH Key
  • 将拷贝的公钥复制在key中(title可不写),点击Add SSH Key

测试

在桌面空白处鼠标右键,选择Git Bash Here

#输入完且回车后，再输入`yes`回车
ssh -T git@github.com

如果出现 Hi! You've successfully authenticated 则配置成功

3.部署到Github Pages

进入bash界面输入

#<user>处填写自己的Github用户名,<email>同理
git config --global user.name "<user>"
git config --global user.email "<email>"

进入博客根目录，找到_config.yml
设置参数 

#<user>处填写自己的Github用户名
url: https://<user>.github.io/
deploy:
    type: git
    repo: git@github.com:<user>/<user>.github.io.git
    branch: master
    message:

安装部署插件

pnpm add hexo-deployer-git

将自己的博客部署到Github上

hexo d -g

三.将网站托管到Vercel上(可以加速页面访问)

  • 进入vercele网页,用GitHub账号登录

  • 点击New project，再点击``,找到自己创建的博客仓库，点击旁边的import

  • 跳转后在Import Git Repository中选择你的Hexo博客仓库

  • 跳转后点击Deploy上传到 Vercel

  • 上传完成点击Goto Dashboard来到项目主页，选择顶部的Settings，在Project Name中更改你网站的名称(域名)

四.我建站遇到的一些问题

1.中文乱码

在博客_config.yml的参数中，若要使用中文，需要修改语言参数

#具体是zh-CN，还是zh-Hans,需要根据主题说明设定。
language: zh-CN

此外还有编码问题

可能你打开_config.yml文件所使用的阅读器默认编码格式非utf-8(Windows自带记事本默认ANSI编码格式)
故而保存后，也是以非utf-8的编码格式保存的，故而乱码

除了_config.yml文件，其他界面，例如文章的md等文件，也要注意编码问题，将其以utf-8编码格式保存。

2.代码高亮异常(代码块显示异常)

建站成功后，博客会自动生成一个hello world文章
但是文章中的代码块区域，会被分为两块，且语言类型提示全部是line，不符合文件md中的效果
自己写了一篇测试文章后，也是同样的问题
通过翻阅各种博客文件，最后在根目录的_config.yml中找到了关键词line-number中的line
对比后发现也确实是这里的问题
经过查阅网上资料，得知高版本的Hexo自带代码高亮和行数显示
这与[主题ParticelX]的代码高亮冲突(转义时会添加部分信息，从而冲突)。
故在博客根目录_config.yml中，将hexo自带高亮关闭
具体修改如下:

highlight:
    enable: false
prismjs:
    enable: false

其实Argvchs大佬在ParticleX主题使用说明提到过这个问题，但不妨碍我瞎啊(悲)

所以，还是建议大家使用主题的时候，仔细阅读相关的主题使用说明，配置好参数。
